# Helm + Kubernetes 本地部署 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 將 metro monorepo 的 server 和 web 打包成 Docker image，透過 Helmfile 管理兩個獨立 Helm chart，部署到 Docker Desktop Kubernetes，並透過 nginx-ingress 以 `metro.local` 和 `api.metro.local` 對外暴露。

**Architecture:** Helmfile 作為頂層 orchestrator，管理 `charts/server` 和 `charts/web` 兩個獨立 Helm chart，部署到 `metro` namespace。Server 以 production multi-stage Docker build 執行 `node dist/index.js`；Web 以 nginx 靜態服務，內部 proxy `/api/` 請求到 server 的 ClusterIP Service DNS。

**Tech Stack:** Docker, Kubernetes (Docker Desktop), Helm 3, Helmfile, nginx-ingress-controller

## Task 2: 更新 Web nginx.conf

**Files:**
- Modify: `../web/nginx.conf`

- [ ] **Step 1: 更新 proxy_pass 目標**

將 `../web/nginx.conf` 改為：

```nginx
server {
    listen 80;

    root /usr/share/nginx/html;
    index index.html;

    location /api/ {
        proxy_pass http://metro-server.metro.svc.cluster.local:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

- [ ] **Step 2: 建立 helmfile.yaml**

建立 `devops/helm/helmfile.yaml`：

```yaml
repositories: []

helmDefaults:
  createNamespace: true
  wait: true
  timeout: 120

releases:
  - name: server
    namespace: metro
    chart: ./charts/server
    values:
      - values/common.yaml
      - values/server.yaml

  - name: web
    namespace: metro
    chart: ./charts/web
    values:
      - values/common.yaml
      - values/web.yaml
```

- [ ] **Step 3: 建立 values/common.yaml**

```yaml

domain:
  web: metro.local
  api: api.metro.local
```

- [ ] **Step 4: 建立 values/server.yaml**

```yaml
  tag: local
  pullPolicy: Never

  name: metro
  user: root
  password: ""
```

**Files:**

- [ ] **Step 1: 建立 Chart.yaml**

建立 `devops/helm/charts/server/Chart.yaml`：

```yaml
apiVersion: v2
name: server
description: Metro API server
type: application
version: 0.1.0

```

- [ ] **Step 2: 建立 values.yaml（chart 預設值）**

建立 `devops/helm/charts/server/values.yaml`：

```yaml
replicaCount: 1

image:
  repository: metro-server
  tag: local
  pullPolicy: Never

service:
  port: 3000

db:
  host: mysql.infra.svc.cluster.local
  port: "3306"
  name: metro
  user: root
  password: ""

domain:
  api: api.metro.local
```

- [ ] **Step 3: 建立 templates/configmap.yaml**

建立 `devops/helm/charts/server/templates/configmap.yaml`：

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: metro-server-config
  namespace: {{ .Values.namespace | default .Release.Namespace }}
data:
  DB_HOST: {{ .Values.db.host | quote }}
  DB_PORT: {{ .Values.db.port | quote }}
  DB_NAME: {{ .Values.db.name | quote }}
```

- [ ] **Step 4: 建立 templates/secret.yaml**

建立 `devops/helm/charts/server/templates/secret.yaml`：

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: metro-server-secret
  namespace: {{ .Values.namespace | default .Release.Namespace }}
type: Opaque
stringData:
  DB_USER: {{ .Values.db.user | quote }}
  DB_PASSWORD: {{ .Values.db.password | quote }}
```

- [ ] **Step 5: 建立 templates/deployment.yaml**

建立 `devops/helm/charts/server/templates/deployment.yaml`：

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: metro-server
  namespace: {{ .Values.namespace | default .Release.Namespace }}
spec:
  replicas: {{ .Values.replicaCount }}
  selector:
    matchLabels:
      app: metro-server
  template:
    metadata:
      labels:
        app: metro-server
    spec:
      containers:
        - name: server
          image: "{{ .Values.image.repository }}:{{ .Values.image.tag }}"
          imagePullPolicy: {{ .Values.image.pullPolicy }}
          ports:
            - containerPort: 3000
          envFrom:
            - configMapRef:
                name: metro-server-config
            - secretRef:
                name: metro-server-secret
```

- [ ] **Step 6: 建立 templates/service.yaml**

建立 `devops/helm/charts/server/templates/service.yaml`：

```yaml
apiVersion: v1
kind: Service
metadata:
  name: metro-server
  namespace: {{ .Values.namespace | default .Release.Namespace }}
spec:
  selector:
    app: metro-server
  ports:
    - port: 3000
      targetPort: 3000
  type: ClusterIP
```

- [ ] **Step 7: 建立 templates/ingress.yaml**

建立 `devops/helm/charts/server/templates/ingress.yaml`：

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: metro-server-ingress
  namespace: {{ .Values.namespace | default .Release.Namespace }}
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
spec:
  ingressClassName: nginx
  rules:
    - host: {{ .Values.domain.api }}
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: metro-server
                port:
                  number: 3000
```


## 快速重新部署流程（日常使用）

每次修改 code 後：

```bash
# 重 build 需要的 image
docker build -t metro-server:local ../server
# 或
docker build -t metro-web:local ../web

# 重啟 pod（強制拉新 image）
kubectl rollout restart deployment/metro-server -n metro
# 或
kubectl rollout restart deployment/metro-web -n metro
```
