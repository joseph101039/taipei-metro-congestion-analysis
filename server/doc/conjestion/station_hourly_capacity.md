# 捷運各站每小時運量計算方法

資料來源：
- `../line_capacity/line_service.csv`（TDX 班距，2024-10）
- `../line_capacity/line_capacity.csv`（每列定員）
- 計算結果：`line_hourly_capacity.csv`

---

## 公式

```
每小時通過班次 = 60 ÷ 班距（MinHeadwayMins）

每小時單向最大乘載量 = 每小時通過班次 × 每列定員
```

**「單向」** 指同一方向一個站一小時內所有列車理論上能載運的旅客上限。

---

## TDX 資料結構說明

TDX `/v2/Rail/Metro/Frequency/TRTC` 提供的是**每個 RouteID** 的班距，非整條線的有效班距。
同一條線會有多個 RouteID（子服務），代表不同行駛區間：

| LineNo | RouteID | 說明 |
|--------|---------|------|
| R | R-1 | 主幹服務 |
| R | R-2 | 副服務（部分時段） |
| R | R-3 | 第三服務（較低頻，深夜班距 12–15 min） |
| BL | BL-1 | 主幹服務 |
| BL | BL-2 | 副服務 |
| G | G-1 | 主幹服務 |
| G | G-2 | 尖峰加班服務（僅 PeakFlag=1 有資料） |
| G | G-3 | 小支線 / 低頻區間，12–20 min |
| O | O-1 | 主幹（含迴龍/蘆洲端） |
| O | O-2 | 副服務（結構同 O-1） |
| BR | BR-1 | 唯一服務 |


* RouteId 的行駛區間需比對 `/v2/Rail/Metro/StationOfRoute/TRTC`，以確定哪些站有重疊服務。
- [tdx_台北捷運營運路線.json](../line_capacity/tdx_%E5%8F%B0%E5%8C%97%E6%8D%B7%E9%81%8B%E7%87%9F%E9%81%8B%E8%B7%AF%E7%B7%9A.json)
- [tdx_新北捷運發車班距頻率資料.json](../line_capacity/tdx_%E6%96%B0%E5%8C%97%E6%8D%B7%E9%81%8B%E7%99%BC%E8%BB%8A%E7%8F%AD%E8%B7%9D%E9%A0%BB%E7%8E%87%E8%B3%87%E6%96%99.json)


> **Y（環狀線）** 來自新北捷運 TDX 端點（SrcUpdateTime 2024-04-12），RouteID Y-1。
> ServiceTag 備註：「板橋至新北產業園區班距約 23 分、中和至大坪林班距約 10 分」，
> 代表全線端到端班距約 23 min，但中段（中和↔大坪林）因兩端加班疊加，有效班距約 10 min。

**一個站的有效班距**取決於有幾條 RouteID 通過：
- 只有 R-1 通過的站（如淡水、象山）→ 班距 = R-1 的班距
- R-1 + R-2 都通過的重疊段 → 有效班距 ≈ 各 route 班距的調和平均（理想交錯時為 R-1班距/2）

## 應用到單一站

### 單線站（僅一條 RouteID 通過）

```
站的每小時單向最大乘載 = 60 ÷ 班距(min) × 列車定員
```

例：淡水站（僅 R-1，平日尖峰）
```
= 60 ÷ 6 × 1,860 = 18,600 人/小時/方向
```

### 重疊區段（多條 RouteID 同向通過同一站）

```
有效班距 ≈ 1 ÷ Σ(1/各route班距)
站每小時單向乘載 = Σ(各route每小時班次 × 列車定員)
```

例：R-1 + R-2 均以 6 min 班距通過某站（理想交錯）
```
每小時班次 = 60/6 + 60/6 = 10 + 10 = 20 班
乘載 = 20 × 1,860 = 37,200 人/小時/方向
```

> 實際重疊區段需比對各 RouteID 的行駛路段，TDX 另有 `/v2/Rail/Metro/StationOfRoute/TRTC` 可查各路站序。

### 轉乘站（多條線通過）

```
站每小時總乘載 = Σ 各線(有效班次 × 列車定員) × 方向數
```

例：忠孝復興站（BL + BR，平日尖峰）
```
BL-1: 10 班 × 1,860 × 2方向 = 37,200
BR-1: 30 班 × 260 × 2方向  =  15,600
合計：52,800 人/小時
```

## 與實際進出量對照

```
壅擠率 = 實際進站人次（ridership 表） ÷ 每小時單向最大乘載
```

`ridership` 記錄的是乘客旅次（OD），不是某時刻車廂內的累積人數。
若要計算車廂壅擠率，需依行駛路徑推算各區間的在車人數（in-vehicle load）。

## 所需資料表設計

### 1. `route_headways`（各 RouteID 各時段班距 — 預設值）

此表儲存 TDX 來源的班距資料，作為前端模擬的**預設值**。

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | INT PK | auto increment |
| route_id | VARCHAR(10) | e.g. R-1, BL-2, G-3 |
| line_code | VARCHAR(5) | e.g. R, BL, G, O, BR, Y |
| service_day | ENUM('平日','假日') | 營運日類型 |
| peak_flag | TINYINT | 1=尖峰, 0=離峰/一般 |
| start_time | TIME | 時段起始 (e.g. 07:00) |
| end_time | TIME | 時段結束 (e.g. 09:00) |
| min_headway_min | DECIMAL(4,1) | 最小班距（分鐘） |
| max_headway_min | DECIMAL(4,1) | 最大班距（分鐘） |

> 資料來源：`line_service.csv`（共 82 筆）

### 2. `line_capacities`（各線列車定員）

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | INT PK | |
| line_code | VARCHAR(5) | R, BL, G, O, BR, Y |
| capacity_per_train | INT | 每列車定員人數 |

固定值：

| line_code | capacity_per_train |
|-----------|--------------------|
| R | 1,860 |
| BL | 1,860 |
| G | 1,764 |
| O | 1,764 |
| BR | 260 |
| Y | 480 |

### 3. `route_stops`（既有，各 RouteID 的站序）

用途：判斷哪些站被哪些 RouteID 覆蓋，計算重疊區段的有效班距。

| 欄位 | 型別 | 說明 |
|------|------|------|
| route_id | VARCHAR(10) | e.g. R-1 |
| station_code | VARCHAR(10) | e.g. R28 |
| sequence | INT | 站序 |
| cumulative_distance_km | DECIMAL(6,3) | 累積距離 |

### 4. `ridership`（既有，各站每小時進出量）

用途：提供實際人次，與運量上限對比計算壅塞率。

## 前端壅塞模擬機制

### 預設值載入

前端從 API 取得 `route_headways` 表的預設班距，按選定的日期類型（平日/假日）和時段自動填入。

### 使用者可調整參數

| 參數 | 預設來源 | 調整方式 |
|------|---------|---------|
| 各 RouteID 的班距 (min) | `route_headways.min_headway_min` | slider 或 input，以小時為單位獨立調整 |
| 模擬時間範圍 | TimelineBar 選定時段 | 日期 + 起迄小時 |
| 日期類型 | 自動判斷平日/假日 | 可手動切換 |

### 模擬計算流程

```
1. 使用者選定時間（某日 HH:00）
2. 依日期判斷 service_day（平日/假日）
3. 查 route_headways 取得該時段各 RouteID 的 min_headway_min
   → 使用者可覆蓋此值
4. 對每一站：
   a. 找出通過該站的所有 RouteID（from route_stops）
   b. 各 route 每小時班次 = 60 ÷ headway
   c. 站每小時單向最大乘載 = Σ(各route班次 × capacity_per_train)
5. 取 ridership 該站該小時的實際進站人次
6. 壅塞率 = 實際進站人次 ÷ 每小時單向最大乘載
```

### 前端 API 需求

| API | 用途 |
|-----|------|
| `GET /route-headways?service_day=平日` | 取得所有 RouteID 各時段預設班距 |
| `GET /lines/routes` | 取得各 RouteID 覆蓋的站序（既有） |
| `GET /line-capacities` | 取得各線列車定員 |
| `GET /ridership?date=YYYY-MM-DD&hour=HH` | 取得該小時各站進出量（既有） |
